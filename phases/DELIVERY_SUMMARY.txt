1. PHASE_3.1_breakpoint_calling/call_breakpoints.R
   Purpose: Identify and classify rearrangement breakpoints
   Input: synteny_anchored.tsv, constraint_tree.nwk
   Output: rearrangements_raw.tsv
   Lines: 466
   Features:
     - Detects fusions, fissions, inversions, translocations
     - Confidence interval annotation (±5kb)
     - Species pair iteration with block analysis

2. PHASE_3.2_filtering/filter_rearrangements.R
   Purpose: Apply quality filters to rearrangement calls
   Input: rearrangements_raw.tsv
   Output: rearrangements_confirmed.tsv, inferred, artifact + criteria file
   Lines: 354
   Features:
     - Multi-stage quality filtering
     - Artifact detection (small blocks, single support)
     - Three-category classification
     - Student-adjustable thresholds

3. PHASE_3.3_tree_mapping/map_to_tree.R
   Purpose: Assign rearrangements to phylogenetic branches
   Input: rearrangements_confirmed.tsv, constraint_tree.nwk
   Output: rearrangements_mapped.tsv
   Lines: 285
   Features:
     - Parsimony-based branch assignment
     - Reversion detection
     - Confidence scoring
     - Node relationship mapping

4. PHASE_3.4_branch_stats/branch_statistics.R
   Purpose: Compute per-branch rearrangement rates
   Input: rearrangements_mapped.tsv, constraint_tree.nwk
   Output: rearrangements_per_branch.tsv, branch_stats.csv + figures
   Lines: 397
   Features:
     - Normalization by branch length (Myr)
     - Hotspot identification (>2 SD)
     - Rate histograms
     - Clade-level summaries

5. PHASE_3.5_literature_comparison/compare_literature.R
   Purpose: Validate against published karyotype data

   Lines: 347
   Features:
     - Agreement rate calculation
     - Published data template generation
     - Manual review flagging
     - Comparison logic framework
